🚀 INSTRUCTIONS DE LANCEMENT
═══════════════════════════════════════════════════════════════════════════════

INSTALLATION (une seule fois)
  1. pip install -r backend/requirements.txt
  2. cd frontend && npm install

LANCEMENT (Windows - Recommandé)
  1. Double-clic sur start_backend.bat
  2. Double-clic sur start_frontend.bat
  3. Ouvrir http://localhost:3101

LANCEMENT (Manuel - 3 terminaux)
  Terminal 1: python backend/orchestrator.py
  Terminal 2: python backend/api_server.py
  Terminal 3: cd frontend && npm run dev

🐛 DÉPANNAGE RAPIDE
═══════════════════════════════════════════════════════════════════════════════

PROBLÈME: "Backend offline"
  → Vérifier que api_server.py tourne sur port 5000
  → Tester: curl http://localhost:5000/api/health

PROBLÈME: "Port 5000 already in use"
  → Windows: netstat -ano | findstr :5000
  → Tuer le processus: taskkill /PID <PID> /F

PROBLÈME: "CORS error"
  → Installer: pip install flask-cors
  → Vérifier que flask-cors est dans requirements.txt

PROBLÈME: "Module not found"
  → Installer: pip install -r backend/requirements.txt

PROBLÈME: Données périmées (is_stale: true)
  → Vérifier que orchestrator.py tourne
  → Vérifier les fichiers JSON dans output/
